# Git Commit Workflow

## Rules

- Commit after each Codex session.
- Review every diff before commit.
- Do not commit broken builds.

## Suggested Commands

```bash
git status
git add .
git commit -m "feat: add transaction management"
```

## Commit Examples

- `chore: initialize flutter project structure`
- `chore: add firebase foundation`
- `feat: add category management`
- `feat: add transaction management`
- `feat: add dashboard overview`
- `feat: add mybca notification parser`
- `test: add parser tests`
