# UI Guidelines

## Design System

Use Material 3 with a clean productivity style.

## Layout

- Mobile: bottom navigation or navigation rail depending on screen width.
- Desktop: sidebar navigation with content area.
- Web: responsive layout similar to desktop.

## Core Screens

- Dashboard
- Transactions
- Categories
- Statistics
- Debt & Receivables
- Import CSV
- Receipt OCR
- Settings
- Notification Draft Review

## UI States

Every major screen must support:

- Loading state
- Empty state
- Error state
- Success state

## Forms

- Validate required fields.
- Format IDR amounts clearly.
- Use date/time picker.
- Use confirmation dialog for destructive actions.

## Accessibility

- Use readable font sizes.
- Use semantic labels for important buttons.
- Do not rely only on color to show meaning.

## Desktop UX

- Support keyboard navigation where practical.
- Use wider layouts for tables and charts.
- Avoid mobile-only assumptions.
